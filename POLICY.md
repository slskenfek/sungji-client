# Dungeon Survivors Policy

## Core Concept

- 플레이어는 `궁수`, `전사`, `마법사`, `거너` 중 하나를 선택한다.
- 플레이어는 던전의 방 단위 스테이지에 입장한다.
- 전투는 `자동 공격`과 `키보드 이동 조작` 중심으로 설계한다.
- 몬스터는 웨이브 또는 지속 스폰 방식으로 끝없이 몰려온다.
- 몬스터 처치로 경험치를 획득하고, 경험치 누적으로 레벨업한다.
- 레벨업은 생존력, 화력, 범위, 공격 속도 같은 성장 선택으로 이어진다.
- 각 방의 마지막에는 `최종 보스`가 등장한다.
- 보스를 처치하면 현재 방을 클리어한다.
- 방 클리어 시 장비와 보상을 획득한다.

## Prototype Scope

- 현재 프로토타입은 `이동`, `카메라 추적`, `기본 캐릭터 애니메이션`, `체력바`, `경험치바`, `미니맵`, `룸 UI` 표시를 기본 범위로 둔다.
- 장기 컨셉상 직업군은 네 가지지만, 현재 프로토타입 구현은 `전사` 단일 캐릭터에 집중한다.
- 룸 구조는 추후 방 클리어, 보스 스폰, 보상 선택 UI와 연결한다.
- 현재 구현 우선순위 직업은 `전사` 단일 캐릭터다.
- 전사는 `근접 1칸 공격`과 `자동 기본공격`을 기준으로 설계한다.
- 몬스터는 현재 프로토타입에서 `원거리 공격`을 사용한다.

## UX Rules

- 메인 플레이는 한 손 또는 양손 키보드 조작만으로 성립해야 한다.
- 전투 판단은 수동 에임보다 `이동`, `회피`, `포지셔닝`에 무게를 둔다.
- UI는 생존 상태, 성장 상태, 현재 방 진행도를 항상 즉시 이해할 수 있게 유지한다.

## Maintenance Rules

- 게임 컨셉, 전투 규칙, 성장식, 직업 우선순위가 바뀌면 이 문서를 같은 작업에서 즉시 갱신한다.
- 구현 코드가 바뀌어 정책과 실제 동작이 어긋나면 정책 문서를 최신 구현 기준으로 자동 정리한다.
- 시스템 파일은 `config`, `scene`, `entity`, `ui`, `system` 책임 단위로 분리해 유지한다.
